# グチ投げ デザイン・アニメーション設計

## デザインコンセプト

ポップで少しバカっぽいが、入力体験は落ち着いているUIにする。グチの内容は重くても、画面は軽く楽しい雰囲気にする。

## トーン

- 明るい
- 遊び心がある
- ネガティブになりすぎない
- 結果画面は少し派手
- 入力画面は読みやすさ優先

## カラーフィールド

単色テーマに寄せすぎず、到達地点ごとに雰囲気を変える。

- 近場: 青空、芝生、街
- 国内: 空色、白、アクセント赤
- 海外: 海、飛行機、地球
- 宇宙: 濃紺、星、惑星色
- 海王星: 深い青、シアン、星

## UI要素

- テキストエリアは大きめで、角丸は控えめにする
- ボタンは主要操作が一目で分かる色にする
- 結果カードは使ってよいが、カード内カードにはしない
- アイコンや絵文字を適度に使い、説明文に頼りすぎない

## アニメーション方針

MVPではCSS Animationを基本とする。Framer MotionやThree.jsは必須ではない。追加依存を増やさずに十分な演出ができる場合はCSSで実装する。

今回の演出強化では、絵文字主体ではなく「低ポリゴン風の人物が紙を丸めて投げる」表現を目標にする。

### 実現方針

本格的な3Dモデル、ボーンアニメーション、物理演算は使わない。HTML要素とCSSで多角形パーツを組み合わせ、ポリゴン風の質感を作る。

- 人物は頭、胴体、腕、手、脚を複数の多角形パーツで構成する
- `clip-path: polygon(...)` で低ポリゴン風の面を作る
- 面ごとに少し違う色を付けて立体感を出す
- 腕、胴体、紙は `transform` と `transform-origin` で動かす
- 紙は長方形から丸い紙くずへ変形する
- 紙くずは回転、縮小、放物線移動で空へ飛ばす
- 背景は到達地点に合わせて地上、空、宇宙へ変化させる

Three.jsを使う案も可能だが、1日アプリでは調整コストが高い。まずCSSポリゴン表現で作り、余裕がある場合のみCanvasやThree.jsへ拡張する。

### 投げる演出

流れ:

1. ポリゴン風の人物がグチの紙を両手または片手で持つ
2. 紙に数本の線を描き、グチが書かれているように見せる
3. 紙が折れ、縮み、丸まって紙くずになる
4. 人物が体をひねり、腕を後ろへ引く
5. 腕を振り抜き、紙くずが手から離れる
6. 紙くずが右上へ弧を描きながら飛ぶ
7. 遠距離ほど紙くずが小さくなり、背景が宇宙寄りになる
8. 最後に到達地点の絵文字または惑星表現が見え、結果画面へつなぐ

表現例:

- 人物の顔はポリゴン風の怒り顔にする
- 紙は最初だけ平面で、くしゃっと丸まる瞬間に影を強める
- 投げる瞬間に人物の胴体を少し前傾させる
- 紙くずの軌跡に短い白い線や小さな紙片を出す
- 月以降は星の密度を増やす

### ポリゴン風人物の構成

人物は以下のパーツで構成する。

| パーツ | 表現 |
| --- | --- |
| 頭 | 多角形の顔、怒り眉、単純な目と口 |
| 胴体 | 三角形や台形を重ねた上半身 |
| 腕 | 上腕、前腕、手を別要素に分ける |
| 脚 | 立ち姿を支える簡易パーツ |
| 影 | 足元に楕円影を置いて接地感を出す |

腕は紙を丸める段階と投げる段階で角度が変わる。全体を細かくリアルに動かすより、シルエットの変化が分かることを優先する。

### 紙の変形

紙は以下の見た目を段階的に切り替える。

| 段階 | 見た目 |
| --- | --- |
| 平面 | 白い紙、数本のグチ線 |
| 折り | 角が折れ、横幅が少し縮む |
| 圧縮 | 高さと幅が縮み、影が強くなる |
| 紙くず | 丸い多角形、しわ線、回転 |

CSSだけで完全な形状モーフィングが難しい場合は、複数要素を重ねて透明度とスケールを切り替える。

### 飛行軌跡

紙くずは以下を組み合わせて飛ばす。

- `translateX`
- `translateY`
- `rotate`
- `scale`
- `opacity`

放物線はCSSの単一transformでもよいが、より自然にする場合は親要素で移動、子要素で上下・回転を別々にアニメーションさせる。

## 到達地点別ビジュアル

| 到達地点 | 主な絵文字・記号 | 背景イメージ |
| --- | --- | --- |
| 近所の公園 | 🌳 | 公園、青空 |
| 隣町 | 🏘️ | 街並み |
| 北海道 | 🗾 | 広い空、雪色アクセント |
| 沖縄 | 🌺 | 海、南国 |
| ハワイ | 🏝️ | ビーチ、飛行機 |
| ニューヨーク | 🗽 | 都市、夜景 |
| 月 | 🌕 | 宇宙、星 |
| 火星 | 🔴 | 赤い惑星 |
| 木星 | 🪐 | 大きな惑星、渦 |
| 海王星 | 🔵 | 深宇宙、青い惑星 |

実装時に絵文字が環境依存で崩れる場合は、CSS図形やシンプルなテキストアイコンに置き換えてよい。

## アクセシビリティ

- アニメーション中の状態テキストを表示する
- 結果表示は見出し構造で伝わるようにする
- エラーは色だけでなく文言で表示する
- 動きが苦手なユーザー向けに `prefers-reduced-motion` でアニメーションを短縮または簡略化する
- ポリゴン人物と紙の細かい動きは `aria-hidden` にし、状態テキストで進行中であることを伝える

## 注意点

- グチ本文を結果画面に再表示しない
- 投げる演出内の紙には実際のグチ本文を表示しない。線だけで「書かれている感じ」を表現する
- 画面説明テキストを増やしすぎず、操作そのものが分かるUIにする
- スマホでテキスト、ボタン、演出が重ならないよう余白を確保する
- 演出を凝らしすぎて結果表示まで長くなりすぎないよう、最大4秒程度に抑える
