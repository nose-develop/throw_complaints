# グチ投げ デザイン・アニメーション設計

## デザインコンセプト

ポップで少しバカっぽいが、入力体験は落ち着いているUIにする。グチの内容は重くても、画面は軽く楽しい雰囲気にする。

## トーン

- 明るい
- 遊び心がある
- ネガティブになりすぎない
- 結果画面は少し派手
- 入力画面は読みやすさ優先

## カラーフィールド

単色テーマに寄せすぎず、到達地点ごとに雰囲気を変える。

- 近場: 青空、芝生、街
- 国内: 空色、白、アクセント赤
- 海外: 海、飛行機、地球
- 宇宙: 濃紺、星、惑星色
- 海王星: 深い青、シアン、星

## UI要素

- テキストエリアは大きめで、角丸は控えめにする
- ボタンは主要操作が一目で分かる色にする
- 結果カードは使ってよいが、カード内カードにはしない
- アイコンや絵文字を適度に使い、説明文に頼りすぎない

## アニメーション方針

MVPではCSS Animationを基本とする。Framer Motionは必須ではない。追加依存を増やさずに十分な演出ができる場合はCSSで実装する。

### 投げる演出

流れ:

1. 人型アイコンまたは絵文字が構える
2. 紙くずまたはメモが右上へ飛ぶ
3. 紙が飛ぶ間に背景が地上から空、宇宙へ変わる
4. 到達地点に応じた絵文字または背景で結果へつなぐ

表現例:

- `😡` が `📝` を投げる
- 紙が回転しながら右上へ移動する
- 遠距離では紙が小さくなり、星が増える
- 月以降は背景を宇宙寄りにする

## 到達地点別ビジュアル

| 到達地点 | 主な絵文字・記号 | 背景イメージ |
| --- | --- | --- |
| 近所の公園 | 🌳 | 公園、青空 |
| 隣町 | 🏘️ | 街並み |
| 北海道 | 🗾 | 広い空、雪色アクセント |
| 沖縄 | 🌺 | 海、南国 |
| ハワイ | 🏝️ | ビーチ、飛行機 |
| ニューヨーク | 🗽 | 都市、夜景 |
| 月 | 🌕 | 宇宙、星 |
| 火星 | 🔴 | 赤い惑星 |
| 木星 | 🪐 | 大きな惑星、渦 |
| 海王星 | 🔵 | 深宇宙、青い惑星 |

実装時に絵文字が環境依存で崩れる場合は、CSS図形やシンプルなテキストアイコンに置き換えてよい。

## アクセシビリティ

- アニメーション中の状態テキストを表示する
- 結果表示は見出し構造で伝わるようにする
- エラーは色だけでなく文言で表示する
- 動きが苦手なユーザー向けに `prefers-reduced-motion` でアニメーションを短縮または簡略化する

## 注意点

- グチ本文を結果画面に再表示しない
- 画面説明テキストを増やしすぎず、操作そのものが分かるUIにする
- スマホでテキスト、ボタン、演出が重ならないよう余白を確保する
