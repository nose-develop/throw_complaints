# グチ投げ アプリ概要設計

## 目的

「グチ投げ」は、ユーザーが日頃のグチを入力し、その文字数に応じてグチを遠くへ投げ飛ばすWebアプリである。

ネガティブな内容を扱うが、体験は暗くしない。入力したグチを楽しく手放す演出によって、少しだけ気分が軽くなることを目指す。

## MVPスコープ

1日1アプリとして、初期版では以下を必須範囲とする。

- グチ本文の入力
- 最大500文字の入力制限
- リアルタイム文字数カウント
- 空欄時のエラーメッセージ
- 文字数による到達地点判定
- グチを投げるアニメーション
- 結果表示
- もう一度投げる

余裕があれば以下を追加する。

- SNSシェア用テキスト生成
- 到達地点ごとの背景変更
- 飛距離レベル表示の演出強化

## 技術前提

- フレームワーク: Next.js 16.2.6
- UI: React 19.2.4
- スタイリング: Tailwind CSS 4
- ルーティング: App Router
- データ保存: なし
- デプロイ想定: Vercel

実装前には `node_modules/next/dist/docs/` の該当ドキュメントを確認する。特に、App Routerのファイル構成、`'use client'`、Tailwind CSSの扱いを優先して確認する。

## 情報設計

このアプリは1ページ完結型とし、画面遷移URLは増やさない。アプリ内状態で以下の3状態を切り替える。

- 入力状態
- 投げる演出状態
- 結果状態

グチ本文はブラウザ上の一時的なReact stateでのみ扱い、localStorage、Cookie、サーバー、外部APIには保存しない。

## 成功条件

- ユーザーが迷わずグチを入力して投げられる
- 入力文字数によって結果が明確に変わる
- 投げる演出が短く気持ちよく、結果表示まで自然につながる
- スマホでも文字入力、ボタン操作、結果確認がしやすい
- グチの内容が保存されないことが設計上明確である
