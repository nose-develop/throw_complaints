# グチ投げ 機能設計

## 状態管理

Reactのクライアント状態で以下を管理する。

- `complaint`: 入力中のグチ本文
- `phase`: `input` / `throwing` / `result`
- `errorMessage`: 入力エラー
- `result`: 到達地点、レベル、メッセージ、背景種別などの判定結果

このアプリは入力内容を保存しない。localStorage、Cookie、DB、API送信は使わない。

## 入力機能

### 仕様

- 最大500文字
- 入力文字数をリアルタイム表示
- 改行も1文字として扱う
- 判定時は前後の空白を除いた文字数を使う
- 空白のみは未入力扱い

### バリデーション

投げるボタン押下時に以下を判定する。

- `trim()` 後の文字数が0ならエラー
- 1文字以上ならエラーを消して投げる演出へ進む

## 到達地点判定

文字数に応じて固定テーブルで判定する。

| 文字数 | 到達地点 | 飛距離レベル | 背景種別 | メッセージ |
| --- | --- | --- | --- | --- |
| 1-20 | 近所の公園 | Lv.1 | park | ちょっとだけスッキリしたかも |
| 21-50 | 隣町 | Lv.2 | town | ほどよく遠くへ投げました |
| 51-100 | 北海道 | Lv.3 | japan | 北の大地までグチが飛びました |
| 101-150 | 沖縄 | Lv.4 | island | 南の風に乗って飛んでいきました |
| 151-200 | ハワイ | Lv.5 | overseas | かなり遠くまでグチが飛んでいきました |
| 201-250 | ニューヨーク | Lv.6 | overseas | 海を越えてビルの向こうへ消えました |
| 251-300 | 月 | Lv.7 | space | そのグチ、地球を出ました |
| 301-350 | 火星 | Lv.8 | mars | 赤い惑星まで届きました |
| 351-400 | 木星 | Lv.9 | jupiter | とんでもない重力圏まで飛ばしました |
| 401-500 | 海王星 | Lv.10 | neptune | もう誰にも届かないレベルまで飛ばしました |

## 投げる処理

1. 投げるボタン押下
2. 入力値を検証
3. 文字数から結果を判定
4. `phase` を `throwing` に変更
5. 到達地点に応じたアニメーション時間を待つ
6. `phase` を `result` に変更

## アニメーション時間

遠い到達地点ほど少し長くする。ただしテンポを優先し、最大でも3秒程度に収める。

| レベル | 目安時間 |
| --- | --- |
| Lv.1-Lv.2 | 1000ms |
| Lv.3-Lv.4 | 1300ms |
| Lv.5-Lv.6 | 1700ms |
| Lv.7-Lv.8 | 2200ms |
| Lv.9-Lv.10 | 2800ms |

## もう一度投げる

結果画面のボタン押下で以下を行う。

- `complaint` を空にする
- `result` を初期化する
- `errorMessage` を空にする
- `phase` を `input` に戻す

## SNSシェア機能

余裕があれば実装する。Web Share APIが使える場合は `navigator.share` を使用し、未対応環境では共有テキストをクリップボードにコピーする。

共有テキストにはグチ本文を含めない。

## 非機能要件

- スマホで操作しやすいボタンサイズを確保する
- キーボード操作で入力と送信が可能な構造にする
- アニメーション中に連打で状態が壊れないよう、投げるボタンは入力画面でのみ表示する
- 結果判定ロジックはUIから分離し、テストや修正がしやすい形にする
